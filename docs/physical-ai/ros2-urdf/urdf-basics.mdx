# URDF Basics: Links, Joints, and Robot Modeling

## Understanding URDF Structure

The Unified Robot Description Format (URDF) is an XML-based format that describes robot models in ROS 2. It defines the physical and kinematic properties of a robot, including its links, joints, inertial properties, visual geometry, and collision geometry. Understanding URDF is fundamental to creating robot models for both simulation and real-world applications.

## Core Components of URDF

A URDF file consists of several key elements that define a robot's structure:

1. **Links**: Rigid bodies that represent physical parts of the robot
2. **Joints**: Connections between links that define how they can move relative to each other
3. **Materials**: Visual appearance properties
4. **Inertial Properties**: Mass, center of mass, and moment of inertia data
5. **Visual and Collision Geometry**: How the robot appears visually and how it interacts physically in simulation

## Basic URDF Document Structure

```xml
<?xml version="1.0"?>
<robot name="simple_robot" xmlns:xacro="http://www.ros.org/wiki/xacro">
  <!-- Links -->
  <link name="base_link">
    <!-- Inertial properties -->
    <inertial>
      <mass value="1.0"/>
      <origin xyz="0 0 0" rpy="0 0 0"/>
      <inertia ixx="0.1" ixy="0" ixz="0" iyy="0.1" iyz="0" izz="0.1"/>
    </inertial>

    <!-- Visual properties -->
    <visual>
      <origin xyz="0 0 0" rpy="0 0 0"/>
      <geometry>
        <box size="0.5 0.5 0.5"/>
      </geometry>
      <material name="blue">
        <color rgba="0 0 1 1"/>
      </material>
    </visual>

    <!-- Collision properties -->
    <collision>
      <origin xyz="0 0 0" rpy="0 0 0"/>
      <geometry>
        <box size="0.5 0.5 0.5"/>
      </geometry>
    </collision>
  </link>

  <!-- Joints -->
  <joint name="joint1" type="fixed">
    <parent link="base_link"/>
    <child link="child_link"/>
    <origin xyz="0.5 0 0" rpy="0 0 0"/>
  </joint>

  <link name="child_link">
    <inertial>
      <mass value="0.5"/>
      <origin xyz="0 0 0" rpy="0 0 0"/>
      <inertia ixx="0.05" ixy="0" ixz="0" iyy="0.05" iyz="0" izz="0.05"/>
    </inertial>
    <visual>
      <origin xyz="0 0 0" rpy="0 0 0"/>
      <geometry>
        <cylinder radius="0.1" length="0.2"/>
      </geometry>
    </visual>
    <collision>
      <origin xyz="0 0 0" rpy="0 0 0"/>
      <geometry>
        <cylinder radius="0.1" length="0.2"/>
      </geometry>
    </collision>
  </link>
</robot>
```

## Links: The Building Blocks of Robot Models

Links represent the rigid bodies of a robot. Each link has physical properties that define its behavior in simulation and its visual appearance.

### Link Properties

A complete link definition includes three main components:

1. **Inertial Properties**: Define how the link responds to forces and torques
2. **Visual Properties**: Define how the link appears in visualization tools
3. **Collision Properties**: Define how the link interacts with other objects in simulation

### Inertial Properties

The `<inertial>` element defines the mass properties of a link:

```xml
<inertial>
  <mass value="1.0"/>
  <origin xyz="0 0 0" rpy="0 0 0"/>
  <inertia ixx="0.1" ixy="0" ixz="0" iyy="0.1" iyz="0" izz="0.1"/>
</inertial>
```

- **mass**: The mass of the link in kilograms
- **origin**: The center of mass location relative to the link frame
- **inertia**: The 3x3 inertia matrix (only the upper triangular values are specified)

For simple geometric shapes, you can calculate the inertia values using known formulas:

- **Box**: For a box with mass m and dimensions (x, y, z):
  - ixx = m*(y² + z²)/12
  - iyy = m*(x² + z²)/12
  - izz = m*(x² + y²)/12

- **Cylinder**: For a cylinder with mass m, radius r, and length l:
  - ixx = iyy = m*(3*r² + l²)/12
  - izz = m*r²/2

- **Sphere**: For a sphere with mass m and radius r:
  - ixx = iyy = izz = 2*m*r²/5

### Visual Properties

The `<visual>` element defines how a link appears in visualization tools like RViz and Gazebo:

```xml
<visual>
  <origin xyz="0 0 0" rpy="0 0 0"/>
  <geometry>
    <box size="0.5 0.5 0.5"/>
  </geometry>
  <material name="blue">
    <color rgba="0 0 1 1"/>
  </material>
</visual>
```

- **origin**: The visual origin relative to the link frame
- **geometry**: The shape of the visual element (box, cylinder, sphere, or mesh)
- **material**: The visual appearance properties

### Collision Properties

The `<collision>` element defines the collision geometry for physics simulation:

```xml
<collision>
  <origin xyz="0 0 0" rpy="0 0 0"/>
  <geometry>
    <box size="0.5 0.5 0.5"/>
  </geometry>
</collision>
```

Collision geometry can be simpler than visual geometry for performance reasons, but it must accurately represent the physical boundaries of the link.

## Joints: Connecting Robot Components

Joints define the connections between links and specify how they can move relative to each other. Each joint has a type that determines its degrees of freedom.

### Joint Types

ROS 2 supports several joint types:

1. **Fixed**: No movement allowed (0 DOF)
2. **Revolute**: Single axis rotation (1 DOF)
3. **Continuous**: Unlimited rotation around single axis (1 DOF)
4. **Prismatic**: Single axis translation (1 DOF)
5. **Floating**: 6 DOF (3 translation + 3 rotation)
6. **Planar**: Movement on a plane (3 DOF)

### Joint Definition Structure

```xml
<joint name="joint_name" type="revolute">
  <parent link="parent_link_name"/>
  <child link="child_link_name"/>
  <origin xyz="0.5 0 0" rpy="0 0 0"/>
  <axis xyz="0 0 1"/>
  <limit lower="-1.57" upper="1.57" effort="100" velocity="1.0"/>
  <dynamics damping="0.1" friction="0.0"/>
</joint>
```

- **name**: Unique identifier for the joint
- **type**: The joint type
- **parent/child**: Links connected by the joint
- **origin**: Position and orientation of the joint relative to the parent link
- **axis**: Rotation or translation axis (for revolute/prismatic joints)
- **limit**: Movement constraints (for revolute/prismatic joints)
- **dynamics**: Physical properties like damping and friction

### Joint Limits and Constraints

Joint limits are crucial for realistic robot behavior:

```xml
<limit lower="-1.57" upper="1.57" effort="100" velocity="1.0"/>
```

- **lower/upper**: Position limits in radians (revolute) or meters (prismatic)
- **effort**: Maximum effort in Newtons (prismatic) or Newton-meters (revolute)
- **velocity**: Maximum velocity in m/s (prismatic) or rad/s (revolute)

## Practical URDF Examples

### Simple Two-Link Robot Arm

```xml
<?xml version="1.0"?>
<robot name="simple_arm">
  <!-- Base link -->
  <link name="base_link">
    <inertial>
      <mass value="1.0"/>
      <origin xyz="0 0 0.1" rpy="0 0 0"/>
      <inertia ixx="0.01" ixy="0" ixz="0" iyy="0.01" iyz="0" izz="0.01"/>
    </inertial>
    <visual>
      <origin xyz="0 0 0.1" rpy="0 0 0"/>
      <geometry>
        <cylinder radius="0.1" length="0.2"/>
      </geometry>
      <material name="grey">
        <color rgba="0.5 0.5 0.5 1"/>
      </material>
    </visual>
    <collision>
      <origin xyz="0 0 0.1" rpy="0 0 0"/>
      <geometry>
        <cylinder radius="0.1" length="0.2"/>
      </geometry>
    </collision>
  </link>

  <!-- Upper arm -->
  <link name="upper_arm">
    <inertial>
      <mass value="0.8"/>
      <origin xyz="0 0 0.25" rpy="0 0 0"/>
      <inertia ixx="0.02" ixy="0" ixz="0" iyy="0.02" iyz="0" izz="0.001"/>
    </inertial>
    <visual>
      <origin xyz="0 0 0.25" rpy="0 0 0"/>
      <geometry>
        <cylinder radius="0.05" length="0.5"/>
      </geometry>
      <material name="red">
        <color rgba="1 0 0 1"/>
      </material>
    </visual>
    <collision>
      <origin xyz="0 0 0.25" rpy="0 0 0"/>
      <geometry>
        <cylinder radius="0.05" length="0.5"/>
      </geometry>
    </collision>
  </link>

  <!-- Shoulder joint -->
  <joint name="shoulder_joint" type="revolute">
    <parent link="base_link"/>
    <child link="upper_arm"/>
    <origin xyz="0 0 0.2" rpy="0 0 0"/>
    <axis xyz="0 1 0"/>
    <limit lower="-1.57" upper="1.57" effort="50" velocity="2.0"/>
  </joint>
</robot>
```

## URDF Best Practices

### 1. Proper Kinematic Chain Design
- Ensure all links are connected through joints to form a proper kinematic tree
- The base link should have no parent and serve as the root of the kinematic chain
- Avoid creating loops in the kinematic structure

### 2. Accurate Inertial Properties
- Use realistic mass values based on actual hardware specifications
- Calculate inertia values correctly for the geometry of each link
- Ensure the center of mass is properly positioned

### 3. Efficient Geometry
- Use simple collision geometry for better simulation performance
- Match visual and collision geometry for accurate physics simulation
- Use meshes only when necessary for complex shapes

### 4. Consistent Naming Conventions
- Use descriptive names for links and joints
- Follow a consistent naming pattern (e.g., "arm_shoulder_joint", "gripper_left_finger")
- Maintain consistency between URDF, ROS topics, and TF frames

### 5. Validation and Testing
- Validate URDF files using tools like `check_urdf`
- Test the model in simulation before deploying to real hardware
- Verify that joint limits and ranges are appropriate for the intended application

Understanding these URDF basics is essential for creating accurate robot models that work well in both simulation and real-world applications. The proper use of links, joints, and their properties ensures that your robot behaves realistically and can be effectively controlled by ROS 2 nodes.
